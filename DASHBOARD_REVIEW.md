## 🔄 页面跳转逻辑检查

### 跳转方式
Dashboard 使用**单页面应用模式**，通过侧边栏的 `st.sidebar.radio` 进行页面切换。

### 跳转逻辑
1. **数据浏览页面**：
   - 列表 → 详情：点击"查看详情"按钮，设置 `selected_document`，刷新页面
   - 详情 → 列表：点击"返回列表"按钮，清除 `selected_document`，刷新页面

2. **摄取追踪页面**：
   - 列表 → 详情：点击"查看详情"按钮，设置 `selected_trace`，刷新页面
   - 详情 → 列表：点击"返回列表"按钮，清除 `selected_trace`，刷新页面

3. **查询追踪页面**：
   - 列表 → 详情：点击"查看详情"按钮，设置 `selected_trace`，刷新页面
   - 详情 → 列表：点击"返回列表"按钮，清除 `selected_trace`，刷新页面

4. **摄取管理页面**：
   - 查看文档：点击"查看"按钮，设置 `selected_document`，显示提示信息
   - 用户需要手动切换到"数据浏览"页面查看详情

### 跳转问题修复
- ✅ 已修复：`ingestion.py` 中的 `st.switch_page` 调用已替换为提示信息
- ✅ 跳转逻辑合理：使用 session_state 保持状态，用户体验良好
